## Examples

### When no `user` is given (unauthenticated state)

If no `user` prop is given, then a default icon is rendered (`IconUser24`) and only the passed in `items` are rendered in the dropdown portion of the component.

<UserDropdownDisclosure
	items={[
		{
			icon: <SwingsetTestIcon name="sign-in" />,
			label: 'Sign in',
			onClick: () => window.alert('Clicked "Sign in" option'),
		},
		{
			icon: <SwingsetTestIcon name="sign-out" />,
			label: 'Sign out',
			onClick: () => window.alert('Clicked "Sign out" option'),
		},
	]}
/>

### User with an `image` referencing "github" in the text

The label and description of the dropdown list are determined by the `image` property on the given `user` object. When the `image` property contains the text `"github"`:

- The label reads: "Signed in with GitHub"
- The description contains: the user's `nickname`

<UserDropdownDisclosure
	user={{
		image: 'https://avatars.githubusercontent.com/u/761456?s=200&v=4',
		email: 'salty@thehound.design',
		nickname: 'salty-the-hound',
	}}
	items={[
		{
			href: '#',
			icon: <SwingsetTestIcon name="bookmark" />,
			label: 'Bookmarks',
		},
		{
			href: '#',
			icon: <SwingsetTestIcon name="external-link" />,
			label: 'Account Settings',
		},
		{
			icon: <SwingsetTestIcon name="sign-out" />,
			label: 'Sign Out',
			onClick: () => window.alert('You clicked "Sign Out"'),
		},
	]}
/>

### User with an `image` not referencing "github" in the text

The label and description of the dropdown list are determined by the `image` property on the given `user` object. When the `image` property **does not** contain the text `"github"`:

- The label reads: "Signed in with Email"
- The description contains: the user's `email`
