The `Heading` component should be used for easily styling `<h1>` - `<h6>` elements to conform to the design system.

## Props

In addition to the props described in the table below, this component also accepts any other valid props for heading elements. A couple of caveats:

- The given `className` prop is applied last to the list of class names set on the rendered element. This means that some CSS properties applied with the following props can be overwritten if needed (not recommended).
- If an `id` prop is given, it will be ignored. Use the `slug` prop instead.

| Prop name | Description                                                 | Required? | Default value | Type                                            |
| --------- | ----------------------------------------------------------- | --------- | ------------- | ----------------------------------------------- |
| `level`   | The `h` element level to render                             | Yes       | N/A           | One of: 1, 2, 3, 4, 5, 6                        |
| `size`    | The number of the `display` style to apply                  | Yes       | N/A           | One of: 100, 200, 300, 400, 500                 |
| `slug`    | The value that will be passed as the heading element's `id` | Yes       | N/A           | string                                          |
| `weight`  | The name of the font weight style to apply                  | Yes       | N/A           | One of: "regular", "medium", "semibold", "bold" |
