The `MdxInlineAlert` component is used in long-form MDX content to highlight various message types. Default titles and icons are provided for each `type`. The default title can be overridden via the `title` prop. Please do not override the default title. 

## Props

| Prop name  | Description                                                            | Required? | Default value      | Type                             |
| ---------- | ---------------------------------------------------------------------- | --------- | ------------------ | -------------------------------- |
| `children` | The alert body text to render.                                         | Yes       | undefined          | string                           |
| `type`     | The 'type' which has associated styles and default data (icon, title). | No        | tip                | tip, warning, or note |
| `title`    | An option to override the default title.                               | No        | The type's default | string                           |

<MdxInlineAlert type="note">

### Warning

## Authoring in MDX

Three prebuilt variants are available for use in both docs and tutorials long-form content: `<Tip />`, `<Note />`, & `<Warning />`.

The only available prop to pass in authorable contexts with these specific variants is `title`. _**Note**_: When using Markdown syntax within the body of an alert, there needs to be a space between the opening / closing
tags and the content — similar to the Tabs or Accordion components.

### Guidelines

- Use the default title for each component rather than overriding the title.
- If a custom title must be used, keep the title short, as this will be the most prominent element when users scan the alert.
- Do not end the title with a period
- Alert messages should be short but clear and descriptive enough to explain what is happening. They should guide users on how to prevent or fix the issue.
- Keep messages under 270 characters.
- Text formatting capabilities via Markdown syntax such as inline links, bold, italic, and lists are supported. Please use bold/italics sparingly to ensure consistency.
- Do not use other Markdown elements. All other Markdown elements are possible but discouraged (header, code snippets, etc.).
- Please don't use images within the alerts. Render them below or above the alert instead.

### Example content

```mdx
## Example tutorial file

This is an example of using the `MdxInlineAlert` variants within MDX content.

<Tip>When your alert is just a plain string, you can write it inline.</Tip>

<Note title="My custom title">

Override the alert's default title via the `title` prop.

</Note>

<Note>

**When you use Markdown syntax** within the body of the alert,
it must be separated by spaces.

</Note>

<Warning>

This is another example with [a link](https://www.hashicorp.com/contact-sales/terraform). 
There must be spaces after and before the opening and closing tags.

</Warning>
```
