The `MdxInlineAlert` component is used in long-form MDX content to highlight various message types. Default titles and icons are provided for each 'type'. The default title can be overridden via the `title` prop.

## Props

| Prop name  | Description                                                            | Required? | Default value      | Type                             |
| ---------- | ---------------------------------------------------------------------- | --------- | ------------------ | -------------------------------- |
| `children` | The alert body text to render.                                         | Yes       | undefined          | string                           |
| `type`     | The 'type' which has associated styles and default data (icon, title). | No        | tip                | tip, highlight, warning, or note |
| `title`    | An option to override the default title.                               | No        | The type's default | string                           |

## Authoring in MDX

Four prebuilt variants are available for use in both docs and tutorials long-form content: `<Tip />`, `<Highlight />`, `<Note />`, & `<Warning />`.

The only available prop to pass in authorable contexts with these specific variants is `title`. _**Note**_: When using markdown syntax within the body of an alert, there needs to be a space between the opening / closing
tags and the content — similar to the Tabs or Accordion components.

### Guidelines

- Keep the title short, as this will be the most prominent element when users scan the alert.
- Do not end the title with a period, regardless if you use a custom headline via title=“my title”
- Alert messages should be short but clear and descriptive enough to explain what is happening. They should guide users on how to prevent or fix the issue.
- We recommend keeping messages under 270 characters.
- Text formatting capabilities via markdown syntax such as inline links, bold, italic, and lists are supported. Please use bold/italics sparingly to ensure consistency.
- Please don't use images within the alerts, render them below or above the alert instead.

### Example content

```mdx
## Example tutorial file

This is an example of using the `MdxInlineAlert` variants within MDX content.

<Tip>When your alert is just a plain string, you can write it inline.</Tip>

<Highlight title="My custom title">
	Override the alert's default title via the `title` prop.
</Highlight>

<Note>

**When you use markdown syntax** within the body of the alert,
it needs to be separated by spaces.

- It handles lists okay
- Just no headings

</Note>

<Warning>

This is another example with [a link](https://www.hashicorp.com/contact-sales/terraform). There need to be spaces after and before the
opening / closing tags.

</Warning>
```
