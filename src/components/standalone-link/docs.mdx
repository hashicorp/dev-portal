The `StandaloneLink` component should be used when a link is needed outside of a body of text. It is based on the <a href="https://design-system-components-hashicorp.vercel.app/components/link/standalone" target="_blank" rel="noreferrer noopener">HashiCorp Design System (HDS) Link::Standalone component (opens in new tab)</a>.

ðŸš¨ Guidance on icon sizes to pass into the `icon` prop:

- If `size` set to "small" or "medium", then size 16 icons are recommended
- If `size` set to "large", then size 24 icons are recommended

See `StandaloneLink`'s <a href="https://github.com/hashicorp/dev-portal/blob/main/src/components/standalone-link/types.ts" target="_blank" rel="noreferrer noopener">types.ts file (opens in new tab)</a> for more details on the props this component exposes.

## Examples

These examples show a `SwingsetTestIcon` that is used for the purpose of rendering icons from `@hashicorp/flight-icons` in Swingset pages. In practice, usage of this commponent should look something like this:

```jsx
import { IconArrowRight16 } from '@hashicorp/flight-icons/svg-react/arrow-right-16'

const MyComponent = () => {
	return (
		<StandaloneLink
			href="/"
			icon={<IconArrowRight16 />}
			iconPosition="trailing"
			text="Get Started"
		/>
	)
}
```

### `StandaloneLinkContents`

`StandaloneLink` exports a utility component called `StandaloneLinkContents`. This utility is responsible for displaying the inner contents of `StandaloneLink`. It is useful where the _look_ of `StandaloneLink` is needed but no `<a>` link element is needed, such as a call-to-action that is placed inside of a `CardLink`.
