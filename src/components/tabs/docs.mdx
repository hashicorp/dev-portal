## Tabs Props

| Prop name            | Description                                                                  | Required? | Default value | Type      |
| -------------------- | ---------------------------------------------------------------------------- | --------- | ------------- | --------- |
| `ariaLabel`          | A non-visible label describing the purpose of the Tabs                       | No        | undefined     | string    |
| `ariaLabelledBy`     | The `id` of an element containing a label describing the purpose of the Tabs | No        | undefined     | string    |
| `children`           | At least two `Tab` components, one for each button and panel to render       | Yes       | N/A           | ReactNode |
| `initialActiveIndex` | The index of the tab to show as active on initial render                     | No        | 0             | number    |
| `showAnchorLine`     | Whether or not a full-width border should be shown below the tab buttons     | No        | false         | boolean   |

## Tab props

| Prop name  | Description                                | Required? | Default value | Type      |
| ---------- | ------------------------------------------ | --------- | ------------- | --------- |
| `children` | The content to render within the tab panel | Yes       | N/A           | ReactNode |
| `heading`  | The text to show in the tab button         | Yes       | N/A           | string    |

### With `Tabs` that oveflow their container

There are times when a row of `Tab`s is too wide to fit inside of its containing element. In these cases, a single `select` element is rendered rather than many `Tab` buttons. For each `Tab`, the `select` will have an `option` for it.

<LiveComponent>{`
<Tabs>
  {Array.from(new Array(10)).map((_, index) => {
    return (
      <Tab heading={"Tab #" + (index+1)}>
        <p>Tab {index+1} content: Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Pretium viverra suspendisse potenti nullam ac tortor. Laoreet id donec ultrices tincidunt arcu non sodales neque. Risus at ultrices mi tempus imperdiet. Morbi tristique senectus et netus et malesuada fames.</p>
      </Tab>
    )
  })}
</Tabs>
`}</LiveComponent>

### With `Tabs` nested within a `Tab`

### With group syncing from `TabProvider`

First, second and third tabs should be actively synced based on their matching `group` property.

_Note_: the `TabProvider` must wrap the page where this component is used for 'groups' to function properly.

_Note_: ideally, tab groups should appear in the same order. However, in practice, it is possible for authors to change the order, or vary the number of tabs. Our implementation needs to account for the fact that a tab with a particular `group` value could appear at any place in the array of tab children.

<LiveComponent>{`
<TabProvider>
  <Tabs>
    <Tab heading="First Tab" group="first">
      <p>First tab content</p>
    </Tab>
    <Tab heading="Second Tab" group="second">
      <p>Second tab content</p>
    </Tab>
    <Tab heading="Third Tab" group="third">
      <p>Third tab content</p>
    </Tab>
  </Tabs>
  <Tabs>
     <Tab heading="Second Group Tab" group="second">
      <p>More second group tab content</p>
    </Tab>
    <Tab heading="First Group Tab" group="first">
      <p>More first tab content</p>
    </Tab>
   
    <Tab heading="Third Tab" group="third">
      <p>More third tab content</p>
    </Tab>
  </Tabs>
</TabProvider>
`}</LiveComponent>
