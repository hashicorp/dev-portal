The `<TryHcpCallout />` components render HCP themed callout cards. There are multiple different components to meet various use cases, each composed of similar pieces.

## Components

### `<TryHcpCallout />`

This composition includes a mock UI visual on large viewports. Intended for use in main content areas, such as on product landing pages.

<LiveComponent>
{`
<TryHcpCallout
  productSlug="terraform"
	heading="Terraform Cloud"
	description="Automate infrastructure provisioning at any scale for free."
	ctaText="Try HCP Terraform for free"
	ctaUrl="https://app.terraform.io/public/signup/account"
/>
`}
</LiveComponent>

### `<TryHcpCalloutCompact />`

This composition includes logo and text elements only. Intended for use in width-constrained areas, such as the sidecar area of our sidebar-sidecar layout.

<LiveComponent>
{`
<TryHcpCalloutCompact
  productSlug="terraform"
	heading="HCP Terraform"
	description="Automate infrastructure provisioning at any scale for free."
	ctaText="Try HCP Terraform for free"
	ctaUrl="https://app.terraform.io/public/signup/account"
/>
`}
</LiveComponent>

### `<TryHcpCalloutPrebuilt />`

This component wraps `<TryHcpCallout />`. It handles content for a set of supported `productSlug` value, which allows this component to provide a single-prop `productSlug` props API.

`<TryHcpCalloutPrebuilt />` is intended for use in authorable contexts. In MDX, we expose the component as `<TryHcpCallout />`, and it accepts a single `product` prop, for consistency with other MDX components.

```mdx
## Hello there!

Here's an MDX example of a `TryHcpCallout`.

<TryHcpCallout product="vault" />

### All product examples

<TryHcpCallout product="hcp" />
<TryHcpCallout product="terraform" />
<TryHcpCallout product="packer" />
<TryHcpCallout product="consul" />
<TryHcpCallout product="vault" />
<TryHcpCallout product="boundary" />
<TryHcpCallout product="nomad" />
<TryHcpCallout product="waypoint" />
<TryHcpCallout product="vagrant" />
```
